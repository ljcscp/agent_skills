# 探索模式

> 用户不确定时，不再死磕追问，转而提供可测试的方向。见 L1 第二步"追问策略选择"表。

---

## 触发条件

满足任一即触发：
- 用户说"我不确定"、"我不知道"、"你帮我看看"
- 用户的问题本身是开放探索型（如"我该选哪个方向"）
- 经过 1-2 轮追问后发现用户自身信息不足

---

## 核心规则

- **不再追问到底**，转而提供 2-4 个可测试的方向
- **每个方向必须包含：** 具体行动步骤 + 预期反馈 + 根据反馈如何决策下一步
- **给出一个时间节点**

---

## 输出格式

### 复杂决策版（4 方向完整输出）

适用于：职业规划、技术选型、重大选择

```
【方向 1】：[方向名称]
- 具体行动：[用户明天就能做的事]
- 如果反馈是 X → [结论]
- 如果反馈是 Y → [结论]

【方向 2】：[方向名称]
- （同上结构）

【建议时间节点】：[X 天/周后，根据反馈做最终决策]
```

### 简单探索版（轻量输出）

适用于：用户想快速排除选项、小决策

```
先试试 [最简单的行动]，如果 [预期结果] 就 [A]，否则 [B]。花 [X 时间] 就能有答案。
```

---

## 注意事项

- 简单问题给简版，复杂决策给完整版——不要所有问题都展开 4 方向
- 每个方向必须是用户"明天就能做"的事，不要说"先学 3 个月"
- 时间节点必须具体，不说"过段时间"
